# Contributing to IoMT Ransomware Detection

Thank you for your interest in contributing! This document outlines how to get involved.

## How to Contribute

### Reporting Bugs

Open an [issue](https://github.com/ethanmalavia/IoMT_ransom/issues) with:
- A clear title and description
- Steps to reproduce
- Expected vs. actual behavior
- Your environment (OS, Python version, PyTorch version)

### Suggesting Features

Open an issue tagged `enhancement` describing:
- The use case
- What the feature would do
- Why it would benefit the project

### Submitting Code

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes — keep commits focused and descriptive
4. Ensure existing functionality is not broken
5. Open a Pull Request against `main` with a clear description

## Code Style

- Follow [PEP 8](https://peps.python.org/pep-0008/) for Python code
- Keep functions focused; avoid side effects where possible
- Add docstrings to new public functions and classes

## Development Setup

```bash
git clone https://github.com/ethanmalavia/IoMT_ransom.git
cd IoMT_ransom
pip install -r requirements.txt
```

## Code of Conduct

Be respectful and constructive. This project follows the [Contributor Covenant](https://www.contributor-covenant.org/) code of conduct.
